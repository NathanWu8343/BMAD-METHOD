# 索引文件任務

## 目的

此任務通過掃描所有文件檔案並確保它們在描述中被正確索引，來維護 `docs/index.md` 檔案的完整性和完整性。它處理根級別文件和子資料夾內的文件，將它們分層組織。

## 任務說明

你現在作為文件索引器運作。你的目標是確保所有文件檔案都在中央索引中被正確編目，並對子資料夾進行適當的組織。

### 必需步驟

1. 首先，定位和掃描：
   - `docs/` 目錄和所有子目錄
   - 現有的 `docs/index.md` 檔案（如果沒有則創建）
   - 文件結構中的所有 markdown（`.md`）和文字（`.txt`）檔案
   - 注意資料夾結構以進行分層組織

2. 對於現有的 `docs/index.md`：
   - 解析當前條目
   - 注意現有的檔案引用和描述
   - 識別任何損壞的連結或遺失的檔案
   - 追蹤已索引的內容
   - 保留現有的資料夾部分

3. 對於每個找到的文件檔案：
   - 提取標題（從第一個標題或檔案名稱）
   - 通過分析內容產生簡要描述
   - 創建到檔案的相對 markdown 連結
   - 檢查它是否已在索引中
   - 注意它屬於哪個資料夾（如果在子資料夾中）
   - 如果遺失或過時，準備更新

4. 對於在索引中找到的任何遺失或不存在的檔案：
   - 呈現所有引用不存在檔案的條目列表
   - 對於每個條目：
     - 顯示完整的條目詳細資訊（標題、路徑、描述）
     - 在移除之前要求明確確認
     - 如果檔案被移動，提供更新路徑的選項
     - 記錄決定（移除/更新/保留）以便最終報告

5. 更新 `docs/index.md`：
   - 維護現有結構和組織
   - 為每個子資料夾創建層級 2 部分（`##`）
   - 首先列出根級別文件
   - 添加帶有描述的遺失條目
   - 更新過時條目
   - 僅移除已確認移除的條目
   - 確保整體格式一致

### 索引結構格式

索引應按如下組織：

```markdown
# Documentation Index

## 根文件

### [Document Title](./document.md)

文件目的和內容的簡要描述。

### [Another Document](./another.md)

這裡的描述。

## Folder Name

在 `folder-name/` 目錄內的文件：

### [Document in Folder](./folder-name/document.md)

此文件的描述。

### [Another in Folder](./folder-name/another.md)

這裡的描述。

## Another Folder

在 `another-folder/` 目錄內的文件：

### [Nested Document](./another-folder/document.md)

巢狀文件的描述。
```

### 索引條目格式

每個條目應遵循此格式：

```markdown
### [Document Title](relative/path/to/file.md)

文件目的和內容的簡要描述。
```

### 操作規則

1. 絕不修改已索引檔案的內容
2. 當 index.md 中現有描述足夠時，保留它們
3. 維護索引中任何現有的分類或分組
4. 所有連結使用相對路徑（以 `./` 開頭）
5. 確保描述簡潔但資訊豐富
6. 絕不在沒有明確確認的情況下移除條目
7. 報告任何發現的損壞連結或不一致性
8. 在考慮移除之前，允許對移動檔案的路徑更新
9. 使用層級 2 標題（`##`）創建資料夾部分
10. 按字母順序排列資料夾，根文件列在最前面
11. 在每個部分內，按標題字母順序排列文件

### 處理輸出

任務將提供：

1. 對 index.md 所做變更的摘要
2. 新索引檔案的列表（按資料夾組織）
3. 更新條目的列表
4. 呈現移除的條目及其狀態：
   - 確認的移除
   - 更新的路徑
   - 儘管檔案遺失仍保留
5. 任何發現的新資料夾
6. 任何發現的其他問題或不一致性

### 處理遺失檔案

對於索引中引用但在檔案系統中找不到的每個檔案：

1. 呈現條目：

   ```markdown
   檢測到遺失檔案：
   標題：[Document Title]
   路徑：relative/path/to/file.md
   描述：現有描述
   部分：[根文件 | 資料夾名稱]

   選項：

   1. 移除此條目
   2. 更新檔案路徑
   3. 保留條目（標記為暫時不可用）

   請選擇一個選項 (1/2/3)：
   ```

2. 在採取任何行動之前等待用戶確認
3. 為最終報告記錄決定

### 特殊情況

1. **分片文件**：如果資料夾包含 `index.md` 檔案，將其視為分片文件：
   - 使用資料夾的 `index.md` 標題作為部分標題
   - 將資料夾的文件列為子部分
   - 在描述中註明這是一個多部分文件

2. **README 檔案**：根據內容將 `README.md` 轉換為更具描述性的標題

3. **巢狀子資料夾**：對於深度巢狀的資料夾，維護層次結構但在主索引中限制為 2 級。更深的結構應該有自己的索引檔案。

## 所需輸入

請提供：

1. `docs/` 目錄的位置（預設：`./docs`）
2. 確認對 `docs/index.md` 的寫入權限
3. 任何特定的分類偏好
4. 任何要從索引中排除的檔案或目錄（例如，`.git`、`node_modules`）
5. 是否包含隱藏檔案/資料夾（以 `.` 開頭）
